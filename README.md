# 맛따라

전라남도 구례군의 맛집과 명소를 발견하고 구례 안에서 하루 코스를 추천받는 React 프로토타입입니다.

## 실행

```bash
npm install
npm run dev
```

전체 검증은 `npm run check`로 실행합니다. 이 명령은 파일별 실코드 줄 수, ESLint,
단위·DOM 테스트, 프로덕션 빌드를 차례로 확인합니다.

## 주요 화면

- 홈: 관리 데이터 기반 카테고리 필터와 맛집 피드
- 상세: 사진, 장소 정보, 지도, 길찾기
- 추천 코스: 세 가지 코스와 단계별 지도 동선
- 관리자 센터: 운영 현황, 회원 레벨, 게시물·코스·맛집 제보·카테고리 관리
- 로그인/회원가입: 아이디·비밀번호, 세션 유지, Kakao·Google OAuth 진입
- 마이페이지: 사용자별 게시물 북마크 저장과 모아보기
- 게시물 공유: 링크 복사, 기기 공유, X·Facebook 공유
- 연관 코스 추천: 조회·북마크에서 계산한 선호도를 쿠키에 저장하고 게시물 상세 하단의 관련 코스 순위에만 반영
- 구례 태그 추천: 산수유·릴랙스·로컬푸드·한옥카페·자연 선택 가중치를 localStorage에 저장
- 구례 전용 데이터: 홈·검색·맞춤 추천·추천 코스에서 구례군 밖의 장소를 차단
- 이미지 캐러셀: 피드 자동 전환, 화살표·모바일 스와이프, 상세 사진별 코멘트 연동
- 탐색 UX: 내부 이동 기록 기반 뒤로가기와 좌우 분할 코스 지도·하단 이동 순서
- 맛집 등록: 로그인한 사용자의 검수 요청 폼
- 코스 피드: 8개 구례 코스를 첫 화면에서 사진·장소 설명과 함께 연속 열람
- 다중 사진 업로드: 최대 8장과 사진별 설명·좌표를 Firebase Storage에 저장
- 실시간 댓글: 게시물과 코스별 Firestore 댓글 작성·열람·본인 댓글 삭제
- 저장 코스 보관함: 로그인 사용자의 저장 코스를 마이페이지에서 실시간 조회
- 지도 경로선: 사진 촬영 순서와 코스 이동 순서를 Polyline으로 연결

## Firebase 로그인과 관리자 권한

Firebase Authentication의 이메일·비밀번호 로그인과 Firestore를 사용합니다. 신규 회원은
`users/{uid}` 문서에 `member` 레벨로 생성되며 북마크와 맛집 제보도 사용자 UID를 기준으로
저장됩니다.

`seoulddddd@gmail.com`, `gaingnarim@gmail.com` 계정은 관리자 센터 접근 계정으로 등록되어 있습니다.

2026-07-22 17:05(KST) 이전에 생성된 기존 사용자 문서는 관리자 레벨로 마이그레이션됩니다.
관리자는 관리자 센터에서 회원을 `일반회원`, `에디터`, `관리자`로 변경할 수 있습니다.
에디터는 콘텐츠와 제보를 관리하고, 관리자만 회원 레벨을 변경할 수 있도록
`firestore.rules`에서 권한을 강제합니다.

피드 카테고리 설정은 현재 브라우저 `localStorage`에 저장되며 게시물, 추천 코스, 회원,
제보 데이터는 Firestore에 저장됩니다.
